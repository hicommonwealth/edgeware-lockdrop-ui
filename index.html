Edgeware Lockdrop
Participate in the Edgeware lockdrop from here. You should already have an EDG wallet with a public address. If you need an EDG wallet, see getting started.
Lockdrop contract address
Contract audited by Quantstamp. See the audit report, and verify the code on Etherscan.
EDG address
Your EDG will go to this address. Double-check this is correct when the transaction is submitted.
Selection
If you want to run a validator node, you must select Lock.
Validate at network launch
Select yes to stake your EDG at network launch. This will require you to have a validator node online when the genesis block launches.
Signalled contract address (optional)
You may signal the balance of a smart contract you have created.
Signalled contract nonce (optional)
You must provide the nonce that was used when the contract was created, if you are signaling a contract.
Submit transaction
Select MyCrypto if you are using a Ledger, Trezor, or Safe-T hardware wallet, or an offline keystore file.

You should be prompted by Metamask to sign and send the transaction.

Step 1.

Copy the raw transaction data below for your respective lockdrop action. You will need to paste this into MyCrypto's interface. 


          

          
Step 2.

          

            Visit MyCrypto
            and select your preferred method of transacting.
            Select advanced transaction options and input the
            transaction data above.
          

          
Step 3.

          

            If you are locking, input the amount you wish to
            lock up in the value field on MyCrypto. Signalers
            should not send any value. Finally, send and sign the
            transaction on the MyCrypto interface.
          

        

        

          
Step 1.

          

            Clone the edgeware-lockdrop
            repo locally, refer to the README for questions as well as the structure of the API.
          

          
Step 2.

          

            You will need to create a .env file in the root directory
            of the repo, containing the following data.
            
ETH_PRIVATE_KEY=...
ETH_ADDRESS=...
LOCKDROP_CONTRACT_ADDRESS=...

          

          
Examples

          

            Locking
            
node ./scripts/lockdrop.js --remoteUrl https://mainnet.infura.io --lock --lockLength 3 --lockValue 0.5 --edgeAddress   --isValidator

            Signaling
            
node ./scripts/lockdrop.js --remoteUrl https://mainnet.infura.io --signal  --nonce  --edgeAddress
